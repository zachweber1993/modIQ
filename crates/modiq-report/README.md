# modiq-report

Assessment reporting and presentation for the modIQ platform.

---

## Purpose

`modiq-report` defines the report domain responsible for representing and formatting assessment results.

This crate provides structured report models and presentation components while remaining independent of assessment execution.

The report crate exists to present deterministic assessment outcomes generated by the engine rather than performing assessment itself.

---

## Responsibilities

This crate owns:

- Assessment Reports
- Finding Summaries
- Recommendation Summaries
- Report Formatting
- Traceability Presentation

The report domain organizes assessment output into consistent, explainable formats suitable for presentation.

---

## Non-Responsibilities

This crate does **not** contain:

- Runtime assessment state
- Engineering knowledge
- Rule evaluation
- Version Profile resolution
- Assessment orchestration
- CLI functionality
- Persistence
- Serialization protocols

These concerns belong to their respective crates.

---

## Design Principles

The reporting domain is intentionally:

- Explainable
- Deterministic
- Presentation-focused
- Traceable
- Platform-independent

Reports should faithfully represent assessment outcomes without modifying or reinterpreting them.

---

## Dependencies

`modiq-report` consumes assessment results produced by:

- modiq-engine
- modiq-runtime

It should remain independent of:

- modiq-cli

Future presentation formats may include terminal, JSON, HTML, PDF, or other rendering targets without changing the underlying report model.

---

## Engineering Notes

The reporting domain is responsible for **representing** assessment results, not **producing** them.

Assessment execution remains the responsibility of the engine.

When in doubt:

> Reports communicate deterministic assessment outcomes—they do not create them.
